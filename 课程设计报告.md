# 《Java程序设计》课程设计报告

## 课题名称：自动化数据处理与清洗系统（DataCleanPro）

---

## 一、课题背景与意义

### 1.1 课题背景

在日常工作与学习中，我们经常需要处理大量的Excel、CSV格式的数据文件。这些数据文件普遍存在以下问题：

- **数据杂乱**：不同来源的数据格式不统一，存在多余空格、大小写混乱等问题
- **空值过多**：数据采集过程中经常产生空值，影响后续分析
- **重复冗余**：数据合并时容易产生重复记录
- **格式不统一**：日期、手机号、邮箱等字段格式各异
- **人工统计效率低**：手动处理大量数据耗时耗力，容易出错

针对上述问题，本课题设计并实现了一个自动化数据处理与清洗系统（DataCleanPro），旨在提供一站式的数据处理解决方案。

### 1.2 课题意义

本系统的设计与实现具有以下意义：

1. **提高数据处理效率**：自动化处理替代人工操作，大幅提升工作效率
2. **保证数据质量**：通过规则校验和数据清洗，确保数据的准确性和一致性
3. **降低出错概率**：程序化处理避免了人工操作可能带来的错误
4. **支持批量处理**：能够同时处理多个文件，满足大规模数据处理需求
5. **提供可视化界面**：友好的GUI界面降低了使用门槛

## 二、需求分析

### 2.1 功能需求

#### 2.1.1 文件导入功能
- 支持Excel文件（.xlsx、.xls）导入
- 支持CSV文件导入
- 支持批量文件上传
- 提供文件预览功能

#### 2.1.2 数据清洗功能
- **去重处理**：自动识别并删除重复数据
- **空值处理**：过滤或填充空值
- **格式修正**：去除多余空格、统一大小写
- **异常数据识别**：标记不符合规范的数据

#### 2.1.3 数据验证功能
- 手机号格式验证（11位数字，以1开头）
- 邮箱格式验证
- 必填字段验证
- 数字范围验证
- 自定义正则表达式验证

#### 2.1.4 数据查询功能
- 支持关键字搜索
- 支持多条件筛选
- 支持分页显示
- 支持数据排序

#### 2.1.5 报表导出功能
- 导出清洗后的数据为Excel格式
- 导出验证错误报告
- 导出操作历史记录

#### 2.1.6 系统管理功能
- 任务历史记录
- 操作日志审计
- 用户登录管理

### 2.2 非功能需求

#### 2.2.1 性能需求
- 支持处理10万行以上的数据文件
- 文件导入响应时间不超过5秒
- 数据清洗处理速度不低于1000行/秒

#### 2.2.2 可靠性需求
- 系统运行稳定，异常情况下能够优雅处理
- 数据处理过程中支持事务回滚
- 提供完整的错误日志记录

#### 2.2.3 可扩展性需求
- 支持自定义验证规则
- 支持插件式清洗器扩展
- 模块化设计便于功能扩展

### 4.1 关键技术实现

#### 4.1.1 多线程处理

使用ExecutorService线程池实现异步任务处理，避免GUI冻结：

```java
// 线程池管理
public class TaskManager {
    private ExecutorService executorService;
    
    public Future<?> submitTask(Runnable task) {
        return executorService.submit(task);
    }
}

// SwingWorker实现GUI异步更新
SwingWorker<DataFile, Void> worker = new SwingWorker<>() {
    @Override
    protected DataFile doInBackground() {
        return importService.importFile(file);
    }
    
    @Override
    protected void done() {
        // 更新GUI
    }
};
worker.execute();
```

public class Server {

#### 4.1.4 连接池管理

```java
public class DBConnection {
    private static BlockingQueue<Connection> pool;
    
    static {
        pool = new LinkedBlockingQueue<>(POOL_SIZE);
        for (int i = 0; i < POOL_SIZE; i++) {
            pool.offer(createConnection());
        }
    }
    
    public static Connection getConnection() throws SQLException {
        return pool.poll(5, TimeUnit.SECONDS);
    }
    
    public static void release(Connection conn) {
        pool.offer(conn);
    }
}
```

### 4.2 核心功能实现

#### 4.2.1 文件导入流程

```
用户选择文件 → 验证文件类型 → 选择解析器 → 解析文件内容 
    → 保存文件元数据 → 批量插入数据行 → 记录操作历史
```

#### 4.2.2 数据清洗流程

```
获取原始数据 → 创建清洗管道 → 执行去重清洗 → 执行空值清洗 
    → 执行格式清洗 → 更新数据库 → 记录清洗结果
```

#### 4.2.3 数据验证流程

```
获取验证规则 → 创建验证规则列表 → 遍历数据行 → 逐行验证 
    → 记录验证结果 → 统计错误数量 → 生成验证报告
```

## 六、总结与展望

### 6.1 项目总结

本项目成功设计并实现了一个自动化数据处理与清洗系统，主要完成了以下工作：

1. **实现了完整的数据处理流程**：包括文件导入、数据清洗、数据验证、数据查询、报表导出等功能
2. **采用了合理的架构设计**：MVC分层架构，模块化设计，便于维护和扩展
3. **运用了多种Java核心技术**：
   - 类继承：FileParser抽象类及子类
   - 接口：DataCleaner、ValidateRule接口
   - 多态：ParserFactory工厂模式
   - 多线程：ExecutorService线程池
   - 网络编程：Socket客户端/服务器
   - GUI编程：Swing图形界面
4. **解决了实际问题**：提供了便捷的数据处理工具，提高了工作效率

### 6.2 项目亮点

1. **完整的OOP设计**：合理运用继承、接口、多态等面向对象特性
2. **异步任务处理**：使用SwingWorker和线程池，保证GUI响应流畅
3. **可扩展的架构**：插件式清洗器和验证规则，便于功能扩展
4. **完善的日志系统**：SLF4J + Logback，支持审计追踪
5. **连接池管理**：自定义连接池，提高数据库访问性能

### 6.3 不足与改进

1. **界面美化**：当前界面相对简单，可以进一步优化UI设计
2. **功能完善**：可以增加数据可视化、图表展示等功能
3. **性能优化**：对于超大文件，可以采用流式处理提高性能
4. **安全增强**：可以增加用户权限管理、数据加密等功能
