## 环境准备

### 1. 数据库（可选）

```bash
# 启动 MySQL，执行建表脚本
mysql -u root -p < src/main/resources/db/schema.sql
```

> 可跳过此步骤——MySQL 不可用时系统自动降级 H2 本地存储。

### 2. 编译并运行

```bash
# 方案A：Maven
mvn clean compile
mvn exec:java -Dexec.mainClass="com.datacleanpro.App"

# 方案B：启动脚本（推荐）
start.bat
```

## 演示流程

### 第一步：启动应用

- 运行后依次执行：初始化数据库连接 → 启动 Socket 服务器（后台） → 加载 Swing GUI
- 状态栏显示 `数据库: 已连接`（或 `未连接`，不影响核心功能）

### 第二步：导入数据文件

1. 切换到 **数据导入** 选项卡
2. 点击 **选择文件**，选择 `src/main/resources/sample/sample.csv`
3. 点击 **导入数据**，表格展示 10 行预览
4. 演示要点：异步导入不阻塞界面、自动解析格式

### 第三步：数据清洗

1. 切换到 **数据清洗** 选项卡，文件下拉框选择刚导入的文件
2. 依次点击 **去除重复数据 / 处理空值 / 格式化数据**
3. 观察右侧日志面板输出清洗结果（总行数、影响行数、删除行数）
4. 演示要点：管道模式依次执行多个清洗器

### 第四步：数据验证

1. 切换到 **数据验证** 选项卡，查看预置的 4 条规则
2. 可新增规则（添加规则按钮，支持 5 种类型）
3. 选择文件后点击 **执行验证**
4. 观察结果面板显示通过/失败统计和失败明细
5. 演示要点：插件式规则引擎、自定义正则

### 第五步：数据查询

1. 切换到 **数据查询** 选项卡，文件下拉框选择文件
2. 在搜索框输入关键字（如 `张三`），点击 **搜索**
3. 观察数据行筛选结果
4. 演示要点：多维度搜索、分页

### 第六步：报表导出

1. 切换到 **报表导出** 选项卡，选择文件
2. 点击 **导出数据**，观察导出路径
3. 检查 `storage/reports/` 目录下的 `.xlsx` 文件
4. 也可导出历史记录
5. 演示要点：Apache POI 生成带格式的 Excel

### 第七步：历史记录

1. 切换到 **历史记录** 选项卡，查看所有操作记录
2. 演示要点：审计日志自动记录每次操作

### 第八步：网络功能（可选）

- 应用启动时自动在后台启动 Socket 服务器（端口 8888）
- 使用 `Client` 类可发送 `IMPORT` / `CLEAN` / `VALIDATE` / `EXPORT` 请求
- 通信采用 Jackson JSON 序列化

## 技术要点讲解

### 1. OOP 特性

| 特性 | 代码位置 | 说明 |
|------|----------|------|
| **继承** | `FileParser` → `ExcelParser` / `CsvParser` | 子类实现 `parse()`，复用 `parseWithResult()` |
| **接口** | `DataCleaner`、`ValidateRule` | 定义行为契约，实现可替换 |
| **多态** | `ParserFactory.getParser()`、`ValidationEngine.createRule()` | 运行时根据类型选择实现 |
| **组合** | `CleanPipeline` 组合多个 `DataCleaner` | 管道模式，链式调用 |

### 2. 多线程

- **SwingWorker**：所有按钮操作通过 SwingWorker 异步执行
- **ExecutorService**：网络请求处理使用线程池
- **效果**：导入大文件时进度条动，界面不冻结

### 3. 异常体系

```
DataCleanException（基类）
├── FileFormatException    # 文件格式不支持
├── DataValidateException  # 数据验证失败
├── DatabaseImportException # 数据库操作失败
└── NetworkException       # 网络通信失败
```

### 4. 数据库回退机制

```
MySQL 可用 ──→ 使用 MySQL 连接池
     ↓ 不可用
H2 本地存储 ──→ 自动建表，storage/local/ 持久化
```

### 5. 设计模式

| 模式 | 使用场景 |
|------|----------|
| 工厂模式 | `ParserFactory` 根据不同文件类型返回对应解析器 |
| 管道模式 | `CleanPipeline` 依次执行多个清洗器 |
| 单例模式 | `TaskManager.getInstance()` |
| 模板方法 | `FileParser.parseWithResult()` 定义骨架 |

## 答辩建议

1. **演示顺序**：文件导入 → 数据清洗 → 数据验证 → 报表导出 → 历史记录
2. **重点讲解**：MVC 分层、接口/继承/多态的代码实现，以及数据库降级设计
3. **现场亮点**：
   - 故意断开 MySQL，展示自动切换 H2 的功能
   - 用 `duplicate_data.csv` 展示去重效果（21行 → 去重后减少）
   - 打开 `large_data.csv` 展示异步导入不卡 UI
4. **常见问题准备**：
   - **问**：和市面上数据清洗工具比有什么优势？→ **答**：侧重于展示 Java OOP 设计能力，完整的 MVC 架构、自定义连接池、异常体系
   - **问**：Swing 已经过时为什么还用？→ **答**：课程要求展示 Java GUI 编程，Swing 是 Java 原生最成熟的桌面方案

## 常见问题

| 问题 | 解答 |
|------|------|
| 数据库连接失败？ | 检查 MySQL 服务；或直接跳过，系统自动用 H2 |
| 文件导入失败？ | 确认格式 `.xlsx` / `.xls` / `.csv`，编码 UTF-8 |
| 导出失败？ | 检查 `storage/reports/` 是否存在且有写入权限 |
| 如何看日志？ | `storage/logs/datacleanpro.log`（应用日志） / `audit.log`（审计日志） |
