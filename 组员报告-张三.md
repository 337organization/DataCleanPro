### 2.1 类继承的实现

在文件解析模块中，我设计了抽象基类`FileParser`，并实现了`ExcelParser`和`CsvParser`两个子类：

```java
// 抽象基类
public abstract class FileParser {
    public abstract List<DataRow> parse(File file);
    public abstract String[] getSupportedExtensions();
    
    public boolean validateFile(File file) {
        // 通用的文件验证逻辑
    }
}

// Excel解析器
public class ExcelParser extends FileParser {
    @Override
    public List<DataRow> parse(File file) {
        // 使用Apache POI解析Excel文件
    }
}

// CSV解析器
public class CsvParser extends FileParser {
    @Override
    public List<DataRow> parse(File file) {
        // 使用OpenCSV解析CSV文件
    }
}
```

通过继承，子类复用了基类的通用逻辑，同时实现了各自的解析算法，体现了面向对象的继承特性。

### 2.2 多态的实现

在解析器工厂中，我实现了多态的解析器选择：

```java
public class ParserFactory {
    private static final Map<String, FileParser> PARSERS = new HashMap<>();
    
    static {
        ExcelParser excelParser = new ExcelParser();
        PARSERS.put("xlsx", excelParser);
        PARSERS.put("xls", excelParser);
        PARSERS.put("csv", new CsvParser());
    }
    
    public static FileParser getParser(String fileType) {
        return PARSERS.get(fileType);
    }
    
    public static ParseResult parseFile(File file) {
        String extension = FileUtil.getFileExtension(file);
        FileParser parser = getParser(extension);
        return parser.parseWithResult(file);  // 多态调用
    }
}
```

通过工厂模式，系统能够根据文件类型自动选择合适的解析器，实现了运行时的多态调用。

### 2.3 多线程的实现

使用ExecutorService线程池实现异步任务处理：

```java
public class TaskManager {
    private static TaskManager instance;
    private ExecutorService executorService;
    
    private TaskManager() {
        this.executorService = Executors.newFixedThreadPool(
            ConfigUtil.getThreadPoolSize()
        );
    }
    
    public Future<?> submitTask(Runnable task) {
        return executorService.submit(task);
    }
}
```

在GUI中使用SwingWorker实现异步更新：

SwingWorker<DataFile, Void> worker = new SwingWorker<>() {

### 2.4 网络编程的实现

实现了Socket客户端/服务器模式：

**服务器端：**
```java
public class Server {
    private ServerSocket serverSocket;
    private ExecutorService threadPool;
    
    public void start() {
        serverSocket = new ServerSocket(port);
        while (running) {
            Socket client = serverSocket.accept();
            threadPool.submit(() -> handleClient(client));
        }
    }
    
    private void handleClient(Socket client) {
        TaskRequest request = ProtocolUtil.receiveObject(client);
        TaskResponse response = processRequest(request);
        ProtocolUtil.sendObject(client, response);
    }
}
```

**客户端：**
```java
public class Client {
    public TaskResponse sendRequest(TaskRequest request) {
        connect();
        ProtocolUtil.sendObject(socket, request);
        return ProtocolUtil.receiveObject(socket, TaskResponse.class);
    }
}
```

## 三、遇到的问题与解决方案

### 3.1 BOM字符问题

**问题描述：** PowerShell的`Set-Content -Encoding UTF8`会在文件开头添加BOM（Byte Order Mark）字符，导致Java编译器报错。

**解决方案：** 编写了批量去除BOM的脚本：
```java
$bytes = [System.IO.File]::ReadAllBytes($file);
if ($bytes[0] -eq 0xEF -and $bytes[1] -eq 0xBB -and $bytes[2] -eq 0xBF) {
    $newBytes = $bytes[3..($bytes.Length-1)];
    [System.IO.File]::WriteAllBytes($file, $newBytes);
}
```

### 3.2 依赖版本冲突

**问题描述：** Apache POI和OpenCSV依赖的commons-io版本冲突。

**解决方案：** 在pom.xml中排除旧版本依赖，显式指定新版本：
```xml
<dependency>
    <groupId>com.opencsv</groupId>
    <artifactId>opencsv</artifactId>
    <exclusions>
        <exclusion>
            <groupId>commons-io</groupId>
            <artifactId>commons-io</artifactId>
        </exclusion>
    </exclusions>
</dependency>
```

### 3.3 数据库连接失败

**问题描述：** MySQL密码配置错误导致连接失败。

**解决方案：** 修改App.java，使应用程序在数据库连接失败时能够优雅降级：
```java
try {
    DBConnection.initialize();
} catch (Exception dbEx) {
    LogUtil.warn("数据库连接失败，部分功能将不可用: " + dbEx.getMessage());
}
```
